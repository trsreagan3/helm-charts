See https://github.com/trsreagan3/kbouncer for full docs.

NOTE: kbounce is a deterrent proxy, not a security boundary. Pair with
Kubernetes RBAC + admission webhooks for defense in depth. The proxy
gates and audits agent traffic that flows through it; nothing stops a
caller from bypassing the proxy and hitting the API server directly if
they have network access and credentials.
